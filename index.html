What I help with

  • Application and product security. Review web applications, APIs, and delivery flows for problems that create real exposure—then translate the findings into clear engineering work.
  • DevSecOps and CI/CD. Add practical guardrails around source control, build pipelines, dependencies, secrets, and releases without turning every pull request into a bottleneck.
  • Cloud and infrastructure security. Review infrastructure-as-code and cloud configuration for identity, network, logging, and cost-risk problems before they become incidents.
  • AI, agent, and MCP security. Threat-model agentic systems, assess tool access and data boundaries, and build protections around prompt injection, unsafe tool use, and supply-chain risk.
  • Security signal, not security noise. Tune scanners and triage processes so the team spends time on exploitable paths and meaningful fixes—not thousands of unactionable findings.

Founders, engineering leaders, software agencies, and product teams that need a security partner who understands how software is actually built and shipped.
