### What I work on

I build tooling and ship pull requests that make security work for engineering teams without slowing them down. Three focus areas:

- **DevSecOps** — CI/CD hardening, SAST/DAST/SCA pipelines, reachability analysis, supply-chain controls, contextual vulnerability triage.
- **FinOps for Security** — cost-aware scanner orchestration, scan-budget controls, finding deduplication, signal-over-volume risk scoring.
- **AI / LLM Security** — agent guardrails, prompt-injection defense, MCP server hardening, AI-codegen vulnerability patterns, model supply-chain integrity.

### Open-source focus

I'm actively contributing pull requests across these areas. Expect PRs that:

- Add reachability-aware vulnerability scoring to scanners and SCA tools.
- Harden CI/CD workflows against supply-chain compromise (pinning, OIDC, least-privilege tokens).
- Add AI-codegen security linting rules to Semgrep, ESLint, and friends.
- Introduce cost controls and budget guards to security tooling.
- Patch MCP servers and agent frameworks against prompt injection and tool-abuse paths.
